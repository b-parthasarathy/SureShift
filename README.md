# SureShift – AI-Driven Weekly Income Protection for Food Delivery Partners

## 🧠 Problem Context

India’s gig economy depends heavily on delivery partners who operate in highly volatile environments. External disruptions such as heavy rainfall, extreme heat, and pollution directly reduce their working hours and, consequently, their income.

While traditional insurance models exist, they are:
- Reactive (claim-based)
- Slow (manual verification)
- Not aligned with daily/weekly earning cycles

Additionally, parametric systems introduce a new risk:

### ⚠️ Critical Challenge: System Exploitation
Gig workers may attempt to manipulate parametric triggers using GPS spoofing tools to falsely appear in disruption zones and claim payouts without actual income loss.

This creates a **trust gap** in automated insurance systems.

---

## 💡 Our Solution: SureShift

SureShift is an AI-powered parametric income protection platform that transforms insurance from a reactive claim system into a **real-time, event-driven safety net**.

The system:
- Monitors environmental disruptions continuously  
- Evaluates real-world impact on worker activity  
- Automatically triggers payouts  
- Validates authenticity using multi-sensor intelligence  

---

## 🚀 Core Innovation Pillars

### 1. Dynamic Income-Based Payout Engine
Unlike fixed compensation systems, SureShift models **actual income loss**.

#### How it works:
- Estimate daily earning baseline (₹600–₹1000)
- Detect disruption duration and severity
- Calculate % of working time lost

#### Example:
- Expected earning: ₹800/day  
- Disruption impact: 50% work loss  
→ Payout ≈ ₹400  

👉 This aligns payouts with **real economic impact**, not arbitrary values.

---

### 2. Multi-Layer Fraud Detection Engine (Key Differentiator)

#### Problem:
GPS spoofing can falsely trigger payouts.

#### Solution:
SureShift introduces a **cross-sensor validation system**:

Inputs:
- GPS location  
- Accelerometer data  
- Gyroscope data  
- Motion patterns  

#### Validation Logic:
- GPS movement must match physical movement  
- Static sensors + moving GPS → flagged anomaly  
- No real activity during disruption → rejected claim  

👉 This ensures:
- Tamper-resistant system  
- Authentic payout distribution  
- Scalable trust model  

---

### 3. Workability Score (AI-Based Trigger Intelligence)

Instead of relying on a single factor (e.g., rainfall), SureShift computes a **Workability Score**:
Workability Score = f(rainfall, temperature, AQI, time-of-day)


#### Behavior:
- Score drops below threshold → worker cannot operate effectively  
- Trigger activates automatically  

👉 This moves from:
❌ Single-condition triggers  
→ ✅ Context-aware decision system
